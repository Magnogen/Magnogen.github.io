Computer science: Network policies

How to make a good network policy

A network policy is used to show the users of the network the rules of using the network. If you want to make a policy for your network, you can address the problems of the following scenarios:

• A user accidentally installs a virus onto the network

• A user attempts to change the configuration of the network

• A user uses a node on the network for unintended purposes

• A user brings in a virus onto the network via external drive

• A user forgets to log off of their computer - only for a hacker to come along and use it

• A user gives their password to other employees - the employees can thus tell a hacker

• A user reveals their personal information

• A user bypasses the security software

• A user downloads an unknown attachment



If your policy contains rules that people can follow to avoid the reprocussions of these events, it is a good policy.

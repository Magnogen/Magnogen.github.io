Sub-programs

Sub-programs are used when you don't want to type out the same code repeatedly throughout your program. There are two different types of sub-program: procedures and functions.

How to use them
In python, you can define a sub-program with the 'def' command. You use it like so:

1| def mySubProgram(arguments):
2|     #Code to be executed when sub-program is called


You can replace the "mySubProgram" with anything you wish to call your sub program. And I'll talk about the "arguments" later on. After line 1, you can type out any code you wish to run when the sub-program is called.

Arguments - what are they?
Arguments are used to you can pass data into the subprogram. This allows it to perform more tasks when compared to when it doesn't have any of these little rascals. As soon as you use an argument in the subprogram, this makes it a function, instead of a procedure. You can 'reference' an argument in the code by treating it as a variable. For example:

1| def test(input1):
2|     print(input1)


The code when written like this:

1| test("Hello World!")


Will output this:

>>>
Hello World!
>>>


Because the string being passed into the sub-program, and the subprogram prints out the string with the 'print' command.

There is a problem though, if the coder doesn't provide any data for the argument, there will be an error (a TypeError to be exact), and the program will break. You can fix this by setting the argument to a default value, this value will then be used as the argument, if there isn't one given. Example:

1| def test(input1="You forgot an argument!"):
2|     print(input1)


Then, if the coder types the following code:

1| test()


The program will output this:

>>>
You forgot an argument!
>>>


What does 'return' do?
The keyword "return" is used so that, when called, the program will output a responce back to the place it was called from. For example:

1| def returnTheNumberFive():
2|     return "5"


Then, when you run this code:

1| print(returnTheNumberFive())


This will be the result:

>>>
5
>>>


Docstrings
Docstrings are used when you want to give an explanation of what the sub-program does, you add them straight after the defining line, like so:

1| def triple(num):
2|     """ returns the value in the brackets, multiplied by 3 """
3|     return num * 3


Here, the docstring is given in the triple quote marks (""").

More examples
This sub-program finds the mean value of all the values in the list given.

1| def findMeanOf(inputList):
2|     total = 0
3|     for i in range(1, len(inputList)):
4|         total = total + int(inputList[i])
5|     total = total / len(inputList)
6|     return total


This sub-program converts celsius to fahrenheit.

1| def fahrenheit(temperature):
2|     return (temperature * 9 / 5) + 32


This sub-program greets a person. If the name is not entered, it will just greet everyone.

1| def greet(name="everybody"):
2|     """ Greets a person """
3|     print("Hello " + name + "!")

Classes

Classes are used when you want to bind multiple sub-programs with a similar topic together.

How to use them
In python, you can create a class with the 'class' command. You use it like so:

1| class myClass:
2|     #Subprograms to be binded together


You can replace the "myClass" with anything you wish to call your class. After line 1, you can type out any sub-program you wish to run when the sub-program is called within the class.

How to call a class
But how do you call a sub-program that is referenced within a class? The answer is quite simple, actually. Here is the class:

1| class myClass:
2|     def mySubProgram1():
3|         print("Code!!!")
4|     def mySubProgram2():
5|         print("More code!!!")


And here is the reference:

1| myClass.mySubProgram1() #Calls 'mySubProgram1' sub-program under 'myClass' class
2| myClass.mySubProgram2() #Calls 'mySubProgram2' sub-program under 'myClass' class


Then, once the program is run, it will produce this output:

>>>
Code!!!
More code!!!
>>>
